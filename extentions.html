FSCSS Extensions Hub: Community Modules & Starter Kits

Extensions · #FSCSS #plugins #opensource #modules

FSCSS extensions are plain .fscss files that add new syntax, utilities, or entire UI components. Import them once, use them everywhere — no build step, no JavaScript overhead.

From naming conventions to animated counters and circular progress indicators, the ecosystem grows with every module. Below you'll find some of the most popular extensions to supercharge your styling workflow.

Naming Systems

camelCase & snake_case Mappings

Stop writing hyphenated CSS properties. These lightweight extensions map modern naming conventions to standard CSS — making your stylesheets feel like JavaScript objects or Python modules.

camelCase Utilities

@import((*) from camelCase)
.panel { backgroundColor: #111; borderRadius: 12px; }

Write JavaScript-style property names: backgroundColor, fontWeight, marginTop. The extension maps each token to its CSS equivalent behind the scenes.

snake_case Mappings

@import((*) from snake_case)
.form { border_radius: 12px; background_color: #f5f5f5; }

Pythonic naming: border_radius, font_weight, margin_top. Keep consistency across backend and frontend codebases.

Both extensions are available via CDN and can be imported selectively. The underlying str() mappings transform each token at compile time — zero runtime overhead.

Animated Counter

counterx.fscss — Percentage Counter Animation

Generate complete @keyframes for smooth step-based counters with a single directive. Perfect for progress indicators, statistics, and animated percentages.

@import(exec(_init counterx)) h2:after { content: "..."; animation: count 5s steps(100) forwards; } @keyframes count { @counter-init("Starting... ", "Loading... ", "Complete!") }

The @counter-init directive generates 101 keyframes (0% → 100%) with customizable text prefixes. Works perfectly with steps(100) for tick-by-tick animation — no JavaScript required.

  • Zero JavaScript — pure CSS counter
  • Customizable start, process, and end messages
  • Uses FSCSS arrays to generate percentage keyframes
  • Lightweight: one line creates 100+ frames

    UI Component

    circle-progress.fscss — Animated Circular Progress

    A reusable radial progress indicator built with conic-gradient, CSS masks, and custom properties. Create animated donut charts and progress rings with just a few FSCSS directives.

    @import((*) from circle-progress) @progress-root() @circle-progress(.progress-circle) .p72 { @progress-range(72) } .big { @progress-size(240px) }

    Then in HTML:

    <div class="progress-circle p72 big">72%</div>

    The plugin exposes utility directives for size, stroke thickness, animation speed, and custom colors via CSS variables.

    Size Control

    .large { @progress-size(260px) } .small { @progress-size(120px) }

    Stroke Thickness

    .thin { @progress-stroke(8px) } .thick { @progress-stroke(20px) }

    Animation

    .smooth { @progress-animate(0.8s) }

    Color customization: Override CSS variables for any instance.

    .blue { --progress-color-arc: #4d9fff; --progress-color-glow: rgba(77,159,255,0.45); }

    Import Guide

    Using Extensions in Your Project

    FSCSS extensions are plain .fscss files hosted on the CDN. Import them using either the exec method (legacy) or the modern from syntax.

    Modern Import (v1.1.16+)

    @import((*) from camelCase) @import((counter-init) from counterx) @import((circle-progress, progress-root) from circle-progress)

    Selective imports: only bring what you need. The wildcard (*) imports all directives from the extension.

    exec Method (legacy)

    @import(exec(_init camelCase)) @import(exec(_init counterx))

    Works with all versions. The _init prefix tells the parser to load the extension without specifying the .fscss extension.

    CDN resolution: FSCSS automatically resolves imports against its CDN. Both camelCase and snake_case extensions are available out-of-the-box; community extensions like counterx and circle-progress are resolved via the same mechanism.

    Create Your Own

    Building Custom Extensions

    An FSCSS extension is simply a .fscss file containing fscss methods like str() mappings, @define blocks, or arrays. Share it with your team or publish it to the community.

    /* my-shorthand.fscss */ str(card, "background: white; border-radius: 24px; padding: 20px; box-shadow: 0 8px 20px rgba(0,0,0,0.05);") @define responsive-padding(mobile, desktop) {` padding: @use(mobile); @media (min-width: 768px) { padding: @use(desktop); } `}

    Once saved, import it like any other extension:

    @import((*) from "my-shorthand.fscss") .card-component { card; @responsive-padding(16px, 32px); }

    Extensions turn FSCSS into a modular styling platform. Build once, import anywhere.
