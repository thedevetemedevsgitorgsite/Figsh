FSCSS Project Series: Design Your Own CSS Language

Guides & Tutorials · #FSCSS #custom-syntax #design-systems #opensource

The FSCSS ecosystem is built on one radical idea: you shouldn't adapt to CSS — CSS should adapt to you. Below you'll find the complete series of deep-dives, tutorials, and practical guides to help you reshape your styling workflow from the ground up.

Each article explores a different facet: from naming conventions like camelCase and snake_case, to parameterized design blocks with @define, and even how to build your own micro-language on top of CSS. No build tools required, just FSCSS.

Extensions & Ecosystem

Community Modules & Starter Kits

FSCSS extensions are just .fscss files. The community has already built modules for typography systems, responsive shorthands, and even dark-mode primitives. Below are some of the most popular series-related starter kits you can import today:

camelCase Utilities

@import((*) from camelCase)

.panel { backgroundColor: #111; }

snake_case Mappings

@import((*) from snake_case)

.form { border_radius: 12px; }

Circle-progress.fscss

@import((
      circle-progress,
      progress-root,
      progress-range
      ) from circle-progress)

        @circle-progress(.progress-circle)

counterx.fscss

@import((*) from counterx)

@keyframes count{ @counter-init() }
